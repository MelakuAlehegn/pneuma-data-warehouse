# Architecture Decision Records

Lightweight ADRs — one file per significant decision. Format:

```
# ADR-NNN — <title>

## Context
What problem are we solving? What constraints apply?

## Decision
What did we choose?

## Alternatives considered
What else did we look at and why did we not pick it?

## Consequences
What does this make easier? Harder? What did we accept?
```

ADRs are immutable once merged. If a decision is revisited, write a new ADR that *supersedes* the old one and leave the old one in place — the history matters.
